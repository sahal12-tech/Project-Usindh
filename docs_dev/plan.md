### The 6 Rules:
1. **Create plan first** - This document and specs file created before coding
2. **Read before decide** - Refer to specs before technical decisions
3. **Update after act** - Update findings and progress after each development session
4. **2-action rule** - After every 2 research/browse operations, document findings
5. **Log all errors** - Document all issues with attempts and resolutions
6. **Never repeat failures** - Document solutions to avoid repeating mistakes

1. **First attempt** - Try standard approach, diagnose if fails
2. **Second attempt** - Try completely different approach
3. **Third attempt** - Question assumptions, research similar solutions
4. **After 3 attempts** - Escalate with documentation of all attempts
